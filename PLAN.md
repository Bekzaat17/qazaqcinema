## Деплой — сделан (2026-07-15)

Прод живёт на `https://qazaqcinema.rehubpro.kz`: авто-TLS (Caddy/Let's Encrypt), webhook,
крон бэкапов (03:30, хранение 14 дней), logrotate для `backup.log`. Авто-восстановление
проверено вживую: краш процесса → контейнер сам за ~2 c; рестарт демона Docker → все 6
сервисов сами (политика `unless-stopped` + `docker`/`containerd` enabled в systemd).

## Что осталось

### 1. Мониторинг/алерты — ПЕРВАЯ ОЧЕРЕДЬ

Сейчас слепая зона: если стек ляжет насмерть (например, бот уйдёт в цикл рестартов из-за
битого env или плохого кода — `unless-stopped` от этого не спасает, он будет перезапускаться
вечно), **узнаем об этом от пользователей**. Что нужно:

- **Крон на ХОСТЕ** (не в контейнере — он должен пережить падение всего стека), раз в 5 мин:
  `curl -fsS https://qazaqcinema.rehubpro.kz/api/health` → проверить `status == ok`
  (эндпоинт уже есть, пингует Redis+БД).
- **Алерт слать напрямую в `api.telegram.org`** (curl + `BOT_TOKEN` из `.env.prod`), а НЕ через
  свой бот: он и есть та штука, которая могла лечь. Telegram API внешний — достучимся всегда.
- **Анти-спам:** уведомлять на СМЕНЕ состояния (ok→fail — «лежит», fail→ok — «поднялся»), а не
  каждые 5 минут. Состояние — флаг-файл рядом со скриптом.
- ⚠️ **Чего этот способ не поймает:** падение самого VPS (крон умрёт вместе с ним) и отвал сети.
  Для этого нужен ВНЕШНИЙ пингер — UptimeRobot / Healthchecks.io / BetterStack, у всех есть
  бесплатный тариф. Одно другого не заменяет: свой крон видит «стек болен, сервер жив», внешний —
  «сервера нет». Ставить лучше оба.
- Скрипт и крон живут в системе → добавить шаг в [DEPLOY.md](DEPLOY.md) (как §5/§5.1), иначе при
  переезде на новый сервер потеряется — `git pull` их не принесёт.

## Идеи на будущее (не в MVP)

**Персонализация и история** (общий фундамент — таблица истории просмотров):
- **История просмотров** — новая таблица `watch_history` (`user_id`, `movie_id`, `watched_at`); писать
  в `PlaybackService.deliver` рядом с инкрементом `play_count` (Фаза 13). Фундамент для полок и
  рекомендаций ниже.
- **«Жалғастыру» / недавно просмотренные** — полка на главной из истории просмотров. NB: видео играется
  в Telegram (`protect_content`), не в Mini App → это «недавно смотренные», а не resume-позиция.
- **История подписок/платежей** — раздел в профиле: тариф, дата, способ, срок. Данные уже есть в
  `payment_requests` (аудит-таблица) → нужен `GET /api/me/payments` + UI, без миграций.
- **Избранное / «Кейін көру» (watchlist)** — пометить фильм «посмотреть позже», отдельная полка/раздел.
  Своя таблица (`user_id`, `movie_id`).
- **Персональные рекомендации** — на основе истории просмотров и категорий («смотрели аниме → …»).

**Прочее:**
- Лидерборды / реферальная программа.
- Mini App админки (добавление фильмов из веба; чекбоксы категорий из справочника).
- Множественные каналы-архивы.
- Каталог: сохранение фильтра/сортировки в состоянии, счётчики на чипах категорий (`category_counts`
  уже отдаёт `count`).
